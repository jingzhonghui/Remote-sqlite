## 1. 项目背景与目标

### 1.1 背景

在服务器运维和开发场景中，SQLite 数据库通常部署在远程 Linux/Windows 服务器上。现有工具要么需要将数据库文件下载到本地后操作，要么依赖纯命令行交互，缺少直观的可视化界面，操作效率低且易出错。

### 1.2 目标

开发一款 **跨平台桌面应用**，使用户能够：

- 通过 SSH 安全连接远程服务器
- 在不下载数据库文件的前提下，直接调用远端 `sqlite3` 命令行工具执行操作
- 提供友好的图形界面完成数据库的增删改查
- 提供 SQL 编辑器，支持语句编写与即时执行
- 提供可视化建表向导，降低数据库设计门槛

### 1.3 目标用户

- 后端开发工程师
- 运维工程师
- 数据分析师
- 需要远程管理嵌入式/轻量级数据库的技术人员

## 4. 系统架构概览

```
┌─────────────────────────────────────────────┐
│               桌面应用（GUI 层）              │
│  连接管理  │  数据库管理  │  SQL编辑器  │ 建表向导 │
└─────────────────────┬───────────────────────┘
                      │  内部通信（IPC / 本地进程）
┌─────────────────────▼───────────────────────┐
│              核心服务层（后端逻辑）            │
│  SSH Client  │  sqlite3 命令包装  │  会话管理  │
└─────────────────────┬───────────────────────┘
                      │  SSH 隧道 / exec
┌─────────────────────▼───────────────────────┐
│              远程服务器（Linux / Windows）     │
│                sqlite3 命令行工具             │
│                SQLite 数据库文件              │
└─────────────────────────────────────────────┘
```

**通信流程**：

1. 用户在 GUI 发起操作
2. 核心服务层将操作转换为 `sqlite3` 命令
3. 通过 SSH 连接在远程服务器上执行命令
4. 捕获标准输出并解析为结构化数据
5. GUI 渲染结果

## 5. 功能需求

### 5.1 SSH 连接管理

#### 5.1.1 新建连接

| 字段 | 必填 | 说明 |
|------|------|------|
| 连接名称 | 是 | 自定义标识 |
| 主机地址 | 是 | IP 或域名 |
| 端口 | 是 | 默认 22 |
| 用户名 | 是 | SSH 登录用户 |
| 认证方式 | 是 | 密码 / 私钥文件 / SSH Agent |
| 私钥路径 | 条件必填 | 认证方式为私钥时必填 |
| 密码 | 条件必填 | 认证方式为密码时必填 |
| 跳板机配置 | 否 | 支持多跳 SSH（ProxyJump） |

#### 5.1.2 连接管理

- **F-SSH-01**：支持保存多个连接配置，本地加密存储（AES-256）
- **F-SSH-02**：支持连接的编辑、复制、删除
- **F-SSH-03**：支持连接测试（Test Connection）
- **F-SSH-04**：显示连接状态（已连接 / 断开 / 连接中）
- **F-SSH-05**：支持自动重连（可配置重试次数和间隔）
- **F-SSH-06**：支持 SSH 隧道（端口转发）
- **F-SSH-07**：连接超时可配置，默认 30 秒
- **F-SSH-08**：支持 known_hosts 校验，防止中间人攻击

#### 5.1.3 会话管理

- **F-SSH-09**：支持同时维持多个 SSH 会话
- **F-SSH-10**：会话空闲超时自动断开（可配置）
- **F-SSH-11**：会话断开时提示用户

---

### 5.2 数据库管理

#### 5.2.1 数据库文件操作

- **F-DB-01**：浏览远程服务器目录，选择 `.db` / `.sqlite` / `.sqlite3` 文件打开
- **F-DB-02**：支持手动输入数据库文件路径
- **F-DB-03**：支持创建新的空数据库文件（在远端执行 `sqlite3 <path>` 初始化）
- **F-DB-04**：显示数据库基本信息（文件大小、修改时间、SQLite 版本）
- **F-DB-05**：支持同时打开多个数据库（标签页切换）

#### 5.2.2 对象树（左侧面板）

- **F-DB-06**：展示数据库下所有表（Tables）、视图（Views）、索引（Indexes）、触发器（Triggers）
- **F-DB-07**：支持刷新对象树
- **F-DB-08**：右键菜单：查看结构、重命名、删除、导出

---

### 5.3 数据浏览与增删改查

#### 5.3.1 数据浏览

- **F-CRUD-01**：以表格形式展示表数据，支持分页（每页行数可配置，默认 100）
- **F-CRUD-02**：支持列宽调整、列排序（点击列头）
- **F-CRUD-03**：支持全文搜索和按列过滤
- **F-CRUD-04**：显示总行数

#### 5.3.2 新增数据（Create）

- **F-CRUD-05**：提供表单或内联编辑方式新增一行
- **F-CRUD-06**：输入校验：NOT NULL 约束、数据类型检查
- **F-CRUD-07**：支持批量导入（CSV / JSON 格式）

#### 5.3.3 修改数据（Update）

- **F-CRUD-08**：双击单元格进入编辑模式
- **F-CRUD-09**：编辑后提交前可预览生成的 SQL
- **F-CRUD-10**：支持撤销未提交的修改

#### 5.3.4 删除数据（Delete）

- **F-CRUD-11**：选中行后支持删除，删除前弹出确认对话框
- **F-CRUD-12**：支持多行批量删除

#### 5.3.5 数据导出

- **F-CRUD-13**：支持将查询结果导出为 CSV、JSON、SQL INSERT 语句
- **F-CRUD-14**：支持导出整张表或当前查询结果集

---

### 5.4 SQL 编辑器

- **F-SQL-01**：提供代码编辑器，支持 SQL 语法高亮
- **F-SQL-02**：支持关键字、表名、列名自动补全（IntelliSense）
- **F-SQL-03**：支持多语句执行（以 `;` 分隔）
- **F-SQL-04**：执行结果以表格展示，多结果集分 Tab 显示
- **F-SQL-05**：显示执行时间、影响行数
- **F-SQL-06**：执行历史记录（最近 100 条，可搜索）
- **F-SQL-07**：支持常用 SQL 片段（Snippets）保存与复用
- **F-SQL-08**：支持快捷键：`Ctrl+Enter` 执行、`Ctrl+/` 注释
- **F-SQL-09**：错误信息高亮显示，定位到出错行
- **F-SQL-10**：支持格式化 SQL（Beautify）

---

### 5.5 可视化建表

#### 5.5.1 建表向导

- **F-DDL-01**：提供图形化界面定义表名
- **F-DDL-02**：列定义面板，支持以下属性：
  - 列名
  - 数据类型（TEXT / INTEGER / REAL / BLOB / NUMERIC，下拉选择）
  - 主键（PRIMARY KEY）
  - 自增（AUTOINCREMENT）
  - 非空（NOT NULL）
  - 唯一（UNIQUE）
  - 默认值（DEFAULT）
  - 检查约束（CHECK）
- **F-DDL-03**：支持添加、删除、拖拽排序列
- **F-DDL-04**：支持设置复合主键
- **F-DDL-05**：支持添加外键（FOREIGN KEY）约束，含 ON DELETE / ON UPDATE 策略

#### 5.5.2 索引管理

- **F-DDL-06**：可视化创建索引，选择列、指定唯一性
- **F-DDL-07**：列出表上所有索引，支持删除

#### 5.5.3 预览与生成

- **F-DDL-08**：实时预览生成的 `CREATE TABLE` DDL 语句
- **F-DDL-09**：点击「执行」在远端数据库中创建表
- **F-DDL-10**：支持修改现有表结构（ALTER TABLE，SQLite 限制内）

## 6. 非功能需求

### 6.1 跨平台支持

| 平台 | 要求 |
|------|------|
| Windows 10/11 (x64) | 必须支持 |
| Linux (Ubuntu 20.04+, Debian, CentOS 7+) x64 | 必须支持 |
| macOS 12+ (Intel / Apple Silicon) | 可选支持 |

### 6.2 性能

- **NF-PERF-01**：应用启动时间 ≤ 5 秒
- **NF-PERF-02**：SSH 连接建立时间 ≤ 5 秒（网络正常情况下）
- **NF-PERF-03**：查询 10 万行以内数据，分页加载延迟 ≤ 2 秒
- **NF-PERF-04**：SQL 编辑器对 1000 行 SQL 文件的打开与高亮渲染 ≤ 1 秒

### 6.3 安全性

- **NF-SEC-01**：SSH 密码和私钥使用操作系统 KeyChain / CredentialManager 或 AES-256 加密后本地存储，不以明文保存
- **NF-SEC-02**：SSH 会话通信全程加密（SSH 协议保障）
- **NF-SEC-03**：支持 known_hosts 指纹校验
- **NF-SEC-04**：应用本身不向任何外部服务器上传用户数据

### 6.4 可靠性

- **NF-REL-01**：网络抖动导致 SSH 断连时，自动尝试重连并提示用户
- **NF-REL-02**：SQL 执行异常时不崩溃，错误信息清晰展示
- **NF-REL-03**：应用崩溃时不损坏用户本地配置文件

### 6.5 可维护性

- **NF-MAIN-01**：代码按功能模块分层组织
- **NF-MAIN-02**：核心逻辑需有单元测试，覆盖率 ≥ 60%
- **NF-MAIN-03**：提供中英文 i18n 支持（初版中文优先）

### 6.6 可用性

- **NF-ACC-01**：支持键盘导航
- **NF-ACC-02**：支持亮色 / 暗色主题切换
- **NF-ACC-03**：界面字体大小可调（12px ~ 18px）

### 7.2 交互原则

- 操作不可逆时（删除、DROP TABLE）必须弹出确认对话框
- 长时操作（大数据导入、连接中）显示进度指示器
- 错误信息使用红色提示，成功操作使用绿色/蓝色提示
- 工具提示（Tooltip）覆盖所有图标按钮

## 8. 技术选型建议

> 以下为推荐方案，最终选型由开发团队确定。

### 8.1 方案 A：Electron + React（推荐）

| 层次 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | Electron | 成熟生态、跨平台、开箱即用的 Node.js 集成 |
| 前端 UI | React + TypeScript | 组件化，TypeScript 增强可维护性 |
| UI 组件库 | Ant Design 或 TDesign | 丰富的表格/表单组件 |
| SSH 客户端 | `ssh2`（Node.js 库） | 纯 JS 实现，无需系统 OpenSSH 依赖 |
| 代码编辑器 | Monaco Editor | VS Code 同款，SQL 高亮/补全支持佳 |
| 状态管理 | Zustand / Redux Toolkit | 管理连接和数据库会话状态 |
| 打包 | electron-builder | 支持 Windows NSIS / Linux AppImage/deb |

### 8.2 方案 B：Tauri + React

| 层次 | 技术 | 理由 |
|------|------|------|
| 桌面框架 | Tauri（Rust 后端） | 体积更小、内存更低 |
| 前端 UI | React + TypeScript | 同上 |
| SSH 客户端 | Rust `russh` crate | 需要 Rust 开发能力 |
| 适用场景 | 对安装包体积和内存消耗敏感时 | — |

### 8.3 SSH 命令执行策略

```
应用 ──SSH exec──▶ 远端 shell ──▶ sqlite3 <db_path> <sql>
                                 ──▶ 输出: .mode csv / .mode json
                                 ──▶ 解析结果 → 表格渲染
```

- 使用 `sqlite3` 的 `.mode json` 或 `.mode csv` 输出便于解析
- 长事务通过 `sqlite3` 交互模式（stdin pipe）维持
- 超时控制：单次命令执行超时默认 30 秒

## 10. 风险与限制

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 远端服务器未安装 `sqlite3` | 核心功能不可用 | 启动时检测并提示用户安装路径或上传二进制 |
| SQLite ALTER TABLE 功能受限 | 修改表结构复杂 | 采用"重建表"方案（创建新表→迁移数据→删除旧表→重命名）|
| SSH 网络延迟高时命令执行慢 | 用户体验差 | 增加 loading 状态、结果流式展示 |
| Electron 打包体积大（~100MB） | 安装包大 | 可选 Tauri 方案；或提供免安装便携版 |
| 大数据量（百万行）性能 | 内存占用高 | 强制分页，禁止全量加载 |
| Windows/Linux 路径差异 | 文件浏览器兼容性 | 统一使用 POSIX 路径处理 |
