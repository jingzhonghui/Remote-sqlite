## 1. 概述

### 1.1 文档目的

本文档提供 RemoteSQLite 桌面应用从环境准备到最终部署的完整流程指导，确保开发人员能够快速、准确地完成应用打包与部署。

### 1.2 支持的平台

| 平台 | 架构 | 打包格式 | 支持状态 |
|------|------|----------|----------|
| Windows | x64 | NSIS | ✅ 完整支持 |
| Linux | x64 | AppImage, deb, rpm, tar.gz | ✅ 完整支持 |
| macOS | Intel, Apple Silicon | DMG | ⚠️ 需要 macOS 环境 |

### 1.3 关键技术栈

- **Node.js**: ^18.0.0 (推荐 ^20.x)
- **Electron**: ^28.0.0
- **electron-builder**: ^24.9.1
- **构建工具**: Vite ^5.0.8

## 3. 项目依赖安装

### 3.1 获取源码

```bash
# 克隆项目（如果使用 Git）
git clone <repository-url>
cd remote-sqlite/src

# 或直接解压源码包
unzip remote-sqlite-source.zip
cd remote-sqlite/src
```

### 3.2 安装 Node.js 依赖

#### 3.2.1 使用 npm（推荐）

```bash
# 进入项目目录
cd /path/to/remote-sqlite/src

# 清理旧依赖（可选，首次安装可跳过）
rm -rf node_modules package-lock.json

# 安装依赖
npm install

# 验证安装
ls node_modules | head -20
```

#### 3.2.2 使用 pnpm（备选）

```bash
# 安装 pnpm
npm install -g pnpm

# 安装依赖
pnpm install

# 验证安装
pnpm list
```

#### 3.2.3 依赖安装说明

| 依赖类型 | 安装方式 | 说明 |
|----------|----------|------|
| 生产依赖 | 自动安装 | package.json 中的 dependencies |
| 开发依赖 | 自动安装 | package.json 中的 devDependencies |
| 原生模块 | electron-builder 自动处理 | ssh2、cpu-features 等 |
| 可选依赖 | 忽略 | 如 cpu-features 失败不影响基本功能 |

### 3.3 原生模块处理

`ssh2` 库依赖原生 C++ 模块，需要针对 Electron 版本重新编译：

```bash
# 方法一：使用 electron-builder 自动处理
npm run postinstall
# 或
npx electron-builder install-app-deps

# 方法二：手动重建（如果上述失败）
npx electron-rebuild

# 方法三：特定模块重建
npx electron-rebuild -m . -f -w ssh2

# 验证原生模块
ls -la node_modules/cpu-features/build/Release/
ls -la node_modules/ssh2/build/Release/
```


### 4.4 Vite 配置检查

确认 `vite.config.ts` 配置正确：

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import electron from 'vite-plugin-electron'
import path from 'path'

export default defineConfig({
  plugins: [
    react(),
    electron([
      {
        entry: 'electron/main.ts',
        onstart: (options) => options.startup(),
        vite: {
          build: {
            rollupOptions: {
              external: ['ssh2', 'cpu-features', 'nan'],  // 原生模块
            },
          },
        },
      },
      {
        entry: 'electron/preload.ts',
        onstart: (options) => options.reload(),
        vite: {
          build: {
            lib: {
              entry: 'electron/preload.ts',
              formats: ['cjs'],
              fileName: () => 'preload.cjs',
            },
          },
        },
      },
    ]),
  ],
  resolve: {
    alias: {
      '@': path.resolve(__dirname, './src'),
    },
  },
})
```

## 5. 打包操作

### 5.1 打包前检查

```bash
# 1. 确保在正确目录
pwd  # 应显示 .../remote-sqlite/src

# 2. 清理旧构建
rm -rf dist dist-electron release

# 3. 验证 node_modules
ls node_modules/.bin/electron  # 应存在

# 4. 检查原生模块
node -e "require('ssh2')"  # 不报错即可
```

### 5.2 完整打包流程

#### 5.2.1 方式一：使用 npm scripts（推荐）

```bash
# 完整打包流程
npm run build && npm run electron:build:linux

# 或使用一键脚本（项目中已配置）
bash scripts/build-linux.sh
```

#### 5.2.2 方式二：分步执行

```bash
# Step 1: 构建前端和主进程
npm run build

# Step 2: 重建原生模块（确保兼容 Electron）
npx electron-builder install-app-deps

# Step 3: 复制必要依赖
node scripts/copy-deps.js

# Step 4: 执行打包
npx electron-builder --linux
```

#### 5.2.3 方式三：指定格式打包

```bash
# 仅 AppImage
npx electron-builder --linux AppImage

# 仅 deb
npx electron-builder --linux deb

# 仅 rpm
npx electron-builder --linux rpm

# 仅 tar.gz
npx electron-builder --linux tar.gz

# 同时生成多种格式
npx electron-builder --linux AppImage deb rpm
```

## 6. 安装与部署

### 6.1 Linux 部署

#### 6.1.1 AppImage 部署（推荐）

**特点**：单文件、绿色无需安装、跨发行版兼容

```bash

chmod +x RemoteSQLite-*.AppImage

# 直接运行
./RemoteSQLite-1.0.0.AppImage

# 可选：集成到系统菜单
./RemoteSQLite-1.0.0.AppImage --appimage-extract
sudo cp squashfs-root/usr/share/applications/RemoteSQLite.desktop /usr/share/applications/
sudo cp squashfs-root/usr/share/icons/hicolor/512x512/apps/RemoteSQLite.png /usr/share/pixmaps/
sudo update-desktop-database /usr/share/applications/
```


### 6.3 企业批量部署

#### 6.3.1 Linux 批量安装脚本

```bash
#!/bin/bash
# deploy.sh - 企业批量部署脚本

APP_NAME="RemoteSQLite"
VERSION="1.0.0"
DEB_FILE="RemoteSQLite_${VERSION}_amd64.deb"
INSTALL_DIR="/opt/${APP_NAME}"
LOG_FILE="/var/log/${APP_NAME}-install.log"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

# 检查 root 权限
if [ "$EUID" -ne 0 ]; then
    echo "请使用 root 权限运行此脚本"
    exit 1
fi

log "开始安装 ${APP_NAME} v${VERSION}"

# 备份旧版本（如有）
if [ -d "$INSTALL_DIR" ]; then
    log "备份旧版本..."
    mv "$INSTALL_DIR" "${INSTALL_DIR}.backup.$(date +%Y%m%d%H%M%S)"
fi

# 安装 deb 包
log "安装 deb 包..."
dpkg -i "/path/to/${DEB_FILE}" 2>&1 | tee -a "$LOG_FILE"

# 修复依赖
log "修复依赖..."
apt-get install -f -y 2>&1 | tee -a "$LOG_FILE"

# 验证安装
if command -v ${APP_NAME,,} &> /dev/null; then
    log "安装成功！"
    ${APP_NAME,,} --version
else
    log "安装失败，请检查日志"
    exit 1
fi
```

## 8. 跨平台打包

### 8.1 Linux 打包（WSL）

在 Windows WSL 环境中打包 Linux 版本：

```bash
# 1. 确认 WSL 版本
wsl --version

# 2. 进入 WSL
wsl

# 3. 进入项目目录
cd /mnt/c/path/to/remote-sqlite/src

# 4. 设置 DISPLAY（用于 Electron GUI）

# 5. 安装依赖
npm install

# 6. 执行打包
npm run electron:build:linux
```

### 8.2 Windows 打包（原生）

```powershell
# 使用 PowerShell 或命令提示符
cd path\to\remote-sqlite\src
```

### 8.3 macOS 打包（需 macOS）

### 9.1 GitHub Actions 配置

创建 `.github/workflows/build.yml`：

```yaml
name: Build and Release

on:
  push:
    tags:
      - 'v*'
  workflow_dispatch:

jobs:
  # Linux 构建
  build-linux:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: src/package-lock.json

      - name: Install Linux dependencies
        run: |
          sudo apt-get update
          sudo apt-get install -y libnss3 libgtk-3-0 libgbm1 libasound2 \
            libxss1 libxtst6 libxrandr2 libxcomposite1 libxcursor1 \
            libxdamage1 libxi6 icnsutils graphicsmagick xz-utils

      - name: Install dependencies
        run: cd src && npm ci

      - name: Build Linux packages
        run: cd src && npm run electron:build:linux

      - name: Upload Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: linux-builds
          path: src/release/

  # Windows 构建
  build-windows:
    runs-on: windows-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: src/package-lock.json

      - name: Install dependencies
        run: cd src && npm ci

      - name: Build Windows package
        run: cd src && npm run electron:build:win

      - name: Upload Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: windows-builds
          path: src/release/

  # macOS 构建
  build-macos:
    runs-on: macos-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'
          cache-dependency-path: src/package-lock.json

      - name: Install dependencies
        run: cd src && npm ci

      - name: Build macOS package
        run: cd src && npm run electron:build:mac

      - name: Upload Artifacts
        uses: actions/upload-artifact@v4
        with:
          name: macos-builds
          path: src/release/

  # 创建 Release
  release:
    needs: [build-linux, build-windows, build-macos]
    runs-on: ubuntu-latest
    if: startsWith(github.ref, 'refs/tags/')
    steps:
      - name: Download all artifacts
        uses: actions/download-artifact@v4
        with:
          path: artifacts

      - name: Create Release
        uses: softprops/action-gh-release@v1
        with:
          files: artifacts/**/*
          draft: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

### 9.3 本地多平台构建脚本

```bash
#!/bin/bash

BUILD_DIR="release"

# Linux
echo "[1/3] Building for Linux..."
npm run electron:build:linux

# Windows (需要 WSL 或 Windows)
echo "[2/3] Building for Windows..."
npm run electron:build:win

# macOS (需要 macOS)

  echo "[3/3] Building for macOS..."
  npm run electron:build:mac
else
  echo "[3/3] Skipping macOS (not on macOS)"
fi

echo "=== Build Complete ==="
echo "Output directory: $BUILD_DIR"
ls -lh "$BUILD_DIR"
```

## 10. 常见问题排查

### 10.1 依赖安装问题

#### 问题 1：npm install 失败

```
Error: Cannot find module 'xxx'
```

**解决方案**：

```bash
# 清理缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install
```

#### 问题 2：node-gyp 编译失败

```
gyp ERR! build error
```

**解决方案**：

```bash
# 安装编译工具
sudo apt-get install -y build-essential python3

# 使用 electron-rebuild
npx electron-rebuild

# 或设置 Python 路径
npm config set python /usr/bin/python3
```

#### 问题 3：cpu-features 编译警告

```
Warning: cpu-features not found
```

**说明**：这是警告不是错误，ssh2 在没有 cpu-features 时仍可正常工作（性能稍低）。

### 10.2 打包问题

#### 问题 4：electron-builder 找不到 electron

```
Error: Cannot find electron
```

**解决方案**：

```bash
# 确认 electron 已安装
ls node_modules/electron

# 重新安装
npm install electron --save-dev

# 或全局安装
npm install -g electron
```

#### 问题 5：AppImage 格式打包失败

```
Error: Cannot create AppImage
```

**解决方案**：

```bash
# 安装 fuse
sudo apt-get install -y fuse libfuse2

# 或使用 no-sandbox 模式构建
npx electron-builder --linux --config.appImage.allow_as_root=true
```

#### 问题 6：原生模块未正确打包

```
Error: Cannot find module '../build/Release/cpu-features.node'
```

**解决方案**：

```bash
# 运行 postinstall
npm run postinstall

# 或手动复制依赖
node scripts/copy-deps.js

# 或修改 package.json build.files 配置
```

### 10.3 运行问题

#### 问题 7：应用启动白屏

**排查步骤**：

```bash
# 1. 检查日志
cat ~/.config/RemoteSQLite/logs/main.log

# 2. 检查开发模式是否正常
npm run electron:dev

# 3. 检查 HTML 文件路径
cat dist/index.html
# 确保引用路径正确
```

**解决方案**：

```typescript
// 修改 electron/main.ts 中的路径
if (process.env.VITE_DEV_SERVER_URL) {
  mainWindow.loadURL(process.env.VITE_DEV_SERVER_URL)
} else {
  // 确保路径正确
  mainWindow.loadFile(path.join(__dirname, '../dist/index.html'))
}
```

#### 问题 8：SSH 连接失败

**排查步骤**：

```bash
# 1. 测试 SSH 命令行
ssh user@host

# 2. 检查服务器 sqlite3
ssh user@host "which sqlite3"
ssh user@host "sqlite3 --version"

# 3. 确认防火墙
ssh -p 22 user@host "exit"
```

#### 问题 9：AppImage 无法运行（缺少 FUSE）

```
fuse: device not found, try 'modprobe fuse' or run with --no-sandbox
```

**解决方案**：

```bash
# 安装 FUSE
sudo apt-get install -y libfuse2

# 或使用 --no-sandbox 运行
./RemoteSQLite-1.0.0.AppImage --no-sandbox

# 或提取运行
./RemoteSQLite-1.0.0.AppImage --appimage-extract
./squashfs-root/AppRun
```

### 10.4 性能问题

#### 问题 10：大文件打包时间过长

**优化方案**：

```bash
# 使用 asar 打包（减小体积）
# 修改 package.json
"asar": true

# 排除不必要的文件
"files": [
  "dist/**/*",
  "dist-electron/**/*",
  "!node_modules/**/*",
  "node_modules/ssh2/**/*"
]
```
