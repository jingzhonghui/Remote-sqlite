# GitHub Actions 自动发布指南

## 工作流说明

本项目配置了 GitHub Actions 自动构建和发布工作流，会在每次创建 Tag/Release 时自动构建并上传各平台的安装包。

### 构建产物

| 平台 | 格式 | 说明 |
|------|------|------|
| Windows | `.exe` (NSIS), `.zip` | Windows 10/11 x64 |
| Linux | `.AppImage`, `.deb`, `.rpm`, `.tar.gz` | Ubuntu, Debian, Fedora, CentOS |
| macOS | `.dmg` | macOS 12+ (Intel/Apple Silicon) |

## 发布步骤

### 方法一：使用 Git Tag 发布（推荐）

```bash
# 1. 切换到 src 目录
cd src

# 2. 更新版本号（语义化版本）
# 编辑 package.json 中的 version 字段，例如改为 x.x.x

# 3. 创建 Tag 并推送
git add .
git commit -m "Release vx.x.x"
git tag vx.x.x
git push origin main --tags
```

### 方法二：手动创建 Release

1. 访问 GitHub 仓库页面
2. 点击 **Releases** → **Draft a new release**
3. 选择或创建一个新的 Tag
4. 填写版本信息
5. 点击 **Publish release**

## GitHub Actions 将会自动

1. 在 Windows、Ubuntu、macOS 三个平台并行构建
2. 构建完成后自动上传安装包到 Release Assets
3. 生成构建产物清单

## 注意事项

### 首次设置

1. 确保仓库已关联到 GitHub
2. 首次运行可能需要在 GitHub Actions 中配置 `GITHUB_TOKEN` 权限（默认已启用）

### 自定义配置

如需修改构建配置：

1. **修改构建命令**：编辑 `.github/workflows/release.yml` 中的 `build_cmd`
2. **修改产物文件**：编辑对应平台的 `files` 配置
3. **修改打包格式**：编辑 `src/package.json` 中的 `build` 配置

### GitHub Token 权限

确保仓库设置中启用了：
- `Contents: Read and write` (用于上传 Release Assets)

## 验证发布

发布成功后，在 Release 页面可以看到类似以下内容：

```
Assets
  RemoteSQLite-Setup.exe     ~50 MB
  RemoteSQLite-x.x.x-win.zip ~80 MB
  RemoteSQLite-x.x.x.AppImage ~70 MB
  RemoteSQLite_x.x.x_amd64.deb ~60 MB
  RemoteSQLite-x.x.x.x86_64.rpm ~60 MB
  RemoteSQLite-mac.dmg       ~75 MB
```

## 故障排除

### 构建失败

检查 Actions 日志中的错误信息，常见问题：
- 依赖安装失败：检查 `package-lock.json` 是否最新
- 构建脚本错误：本地运行 `npm run electron:build` 测试

### 无法上传 Assets

确保 `GITHUB_TOKEN` 有写入权限。在仓库 Settings → Actions 中检查。
